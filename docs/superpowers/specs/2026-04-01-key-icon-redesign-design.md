# 钥匙图标重绘设计（仅形状）

## 1. 背景与目标
用户希望重画应用图标中的钥匙形状，参考提供图片中的钥匙轮廓。目标是在保持现有视觉体系稳定的前提下，显著提升钥匙轮廓的识别度和参考一致性。

## 2. 范围与非目标
### 范围
- 仅修改 `resources/AppIcon.tsx` 中钥匙相关 SVG 几何。
- 重建钥匙头、杆身、齿部三层几何路径。
- 保留多层渐变金属风格。
- 保留钥匙头双环结构（外圈+内圈）。

### 非目标
- 不修改应用业务逻辑、状态流、IPC 调用。
- 不修改图标背景底板（现有渐变矩形保持不变）。
- 不新增图标组件或额外资源文件。

## 3. 当前实现简述
当前图标位于 `resources/AppIcon.tsx`，使用 32x32 SVG，包含背景渐变和三层钥匙叠加（`k132`、`k232`、`k332`），并通过分组变换实现视觉动势。

## 4. 设计方案
### 4.1 几何结构
采用“重建整把钥匙比例”的方案：
- 钥匙头：保留双环，但调整为更贴近参考图的扁圆形比例。
- 连接段：缩短头部到杆身过渡区域，使整体更紧凑。
- 杆身：采用更直、更厚的主轴，保证 32x32 尺寸下的可辨识性。
- 齿部：改为右上方向三段阶梯齿，并在末端加入小折角，贴近参考图轮廓。

### 4.2 分层与光影
保留现有三层金属体系，并让三层共享同一几何骨架后做细微偏移：
- 主层：中性银灰（主体体积）。
- 次层：冷色高光（上缘与转折处）。
- 深层：阴影层（下缘与内侧体积）。

齿部的高光沿上缘连续，阴影布置在齿下缘和杆身内侧，避免阶梯齿在小尺寸下“糊成一块”。

### 4.3 变换与构图
- 保持原有整体构图、旋转动势与图标占比。
- 保持钥匙在背景中的视觉中心位置，避免重绘后偏移。

## 5. 数据流与影响面
该改动为纯静态 SVG 重绘：
- 无新增状态、无事件处理变更。
- 无运行时数据流变化。
- 无跨文件依赖改动。

## 6. 错误处理与风险控制
主要风险与控制点：
- 路径闭合问题：确保每层 path 可正确填充，无裂缝。
- 层级遮挡问题：保证三层叠放顺序与 ring 可见性。
- 小尺寸失真：优先保证 32x32 下齿部边缘清晰。
- 渐变引用冲突：沿用现有 id，不新增冲突 id。

## 7. 验收标准
- 仅改动 `resources/AppIcon.tsx`。
- 钥匙齿部呈“右上三阶 + 末端折角”并清晰可辨。
- 双环结构保留且与新头部几何对齐。
- 仍为多层渐变金属风格，而非纯色描边。
- 应用内图标渲染无错位、无破面、无明显锯齿。

## 8. 验证计划
- 在开发界面观察图标 100% 缩放显示。
- 在深浅背景下快速检查边缘对比与层次可读性。
- 对比重绘前后，确认背景与非钥匙元素无变化。

## 9. 实施文件
- `resources/AppIcon.tsx`（唯一实现改动文件）
